So let's take a look at JS.

JS is...

  • Case sensitive
    • thisvariable and ThisVariable are different.
  • camelCase seems an agreed convention
  • starting lower case
  • Objects start with Caps()
  • CONSTANTS in UPPERCASE
  • JS ignores white space
  • JS is Object Oriented (in a loose way)
  • End each statement with semicolon; (though you don't HAVE to)
  • JS vars - local (decalred in function) / global (declared in root of script) - can access it anywhere inc indside functions.
  • Constant const (e.g, MYCONST) - cannot be changed.
  • let - block scope variable (smaller scope than var) - limits scope to a code block - saves having to create other vars
